## 평가 프로토콜(SOP)

### 목적/범위
- NHANES DPQ(PHQ-9) 기반 고정 라벨을 사용하여 OpenAI 모델 응답의 정확도/안정성 평가
- 본 SOP는 `back/research` 디렉터리 내부 사용에 한정

### 데이터 소스
- 입력: `back/research/data/DPQ_L.xpt` (단일 주기)
- 확장: 추가 병합 금지(현 단계). 인구통계/가중치 조인은 보류

### 라벨 정의
- 총점: 유효값(0–3)으로 `DPQ010–DPQ090` 합(0–27)
- 중증도: 0–4 none, 5–9 mild, 10–14 moderate, 15–19 moderately_severe, 20–27 severe
- needsAttention: (총점 ≥ 10) 기준. 보조 규칙으로 `DPQ100 ≥ 2`는 별도 시나리오로 평가

### 전처리
- 특수코드(미응답/모름/거절 등)는 결측 처리 후 합산
- 결측 허용: 원칙적으로 9문항 모두 응답. 결측 존재 시 해당 샘플은 평가셋에서 제외(현 단계)

### 분할
- 계층화(중증도 기준) 80:20, 랜덤 시드 고정(예: 20250927)
- 분할 결과를 CSV로 저장: `back/research/data/splits/{date}/train.csv`, `test.csv`

### 평가 지표
- riskLevel: 정확도, macro-F1, 혼동행렬
- needsAttention: 정확도, F1
- 안정성: JSON 파싱 성공률(후속 단계)

### 베이스라인
- 규칙 기반과의 비교(문서화): 기준 A(총점≥10), 기준 B(총점≥10 OR DPQ100≥2)

### 재현성
- 입력 파일 해시 기록(SHA-256), 분할 시드/버전 기록, 노트북 커밋 해시 기록
